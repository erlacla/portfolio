  • Should probably start any project with just the root files: index.html , main.js, and main.css.
  • "User errors are in fact also programmer errors, because the program should be designed in a way that prevents the user from making the error."
  • Stack trace = goes function by function to get to error.
  • Warning = a error but not enough to make the program crash.
  • It's good for errors to be loud. "Failing silently makes errors difficult to spot and longer to track down."
  • Linting = helps find js errors or sloppy programming.
  • "Passing a lint test is no guarantee that your code is correct, but it will mean it will be more consistent and less likely to have problems."
  • Determine browser support for a feature with feature detection.
  • Use breakpoints to debug or use alerts or the console
  • "ReferenceError is thrown when a reference is made to an item that doesn’t exist. For example, calling a function that hasn't been defined."
  • "TypeError is thrown when there’s an error in the type of value used; for example, a string is used when a number is expected."
  • You can make your own error message with 'throw'
  • Use catch, try, and finally to see if the code runs properly.
  • Boolean tests help keep code solid, not brittle.
  • TDD = test-driven development. Write tests first, then code, then refractor. Test every time.
