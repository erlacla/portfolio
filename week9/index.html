  • In a browser the global object is the window object.
  • BOM = Browser object model.
  • In BOM we find the browser being used to view a page, the dimensions of the screen it is viewed on, and which pages have been visited before the current page.
  • Every browser window, tab, popup, frame, and iframe has a window object.
  • Node.js probably won’t have a window object, but will still have a global object.
  • Global variables are variables created without using the const, let, or var keywords.
  • parseInt() and isNaN() are actually methods of the global object, which in a browser environment makes them methods of the window object.
  • alert , confirm() , and prompt() are all methods of the window object, and open dialogs that halt the execution of the program.
  • You should refer to global variables without using the window object: window.x === x; UNLESS you need to check if a global variable has been defined.
  • Method of the Number object: Number.parseInt(4.2); << 4 Number.isNaN(4.2); << false

  • Cookies take the form of a text file that contain a list of name/value pairs separated by semicolons, and can be used for personalizing a user’s browsing experience, storing user preferences, keeping track of user choices (such as a shopping cart), authentication and tracking users.
  • To read the cookie (see the current contents): document.cookie: << "name=Batman; hero=true; city=Gotham"
  • Cookies expire by default when the page is closed but you can set an expiration date if you want.
  • For security reasons access to cookies is limited.
  • Cookies can be a bother. Some choose to use a library like cookies.js or jsCookies.
  • Times up!: window.setTimeout( () => alert("Time's Up!"), 3000);
  • You can use the alerts ID to stop the alert before it's finished.
  • setInterval() makes the command repeat again and again after the time you set.
  • Avoid using the "this" keyword for timing methods.
  • It is typically better to use CSS for any animation effects, although there may be times when JavaScript might be the better solution.
  • data- attribute: start with data-, Contain only lowercase letters, numbers, hyphens, dots, colons or underscores, and include an optional string value. data-powers = 'flight superSpeed' data-rating = '5' data-dropdown data-user = 'DAZ' data-max-length = '32'
  • div id='hero' data-powers='flight superSpeed'> Superman div const superman = document.getElementById('hero'); const powers = superman.dataset.powers; 'flight superSpeed'
  • Notice that the data- prefix is dropped. To access the attribute, powers is used as if it’s a property of the dataset object. If a data- attribute’s name contains hyphens, they are replaced with camel-case notation, so data-max-length would be accessed using dataset.maxLength .

  • The Web Storage API provides a useful way of storing various types of information on a user’s computer without the restriction of cookies.

  • Web workers: const worker = new Worker('task.js'); This function takes the name of another JavaScript file as an argument. It allows javascript to continue running in the background so the website will continue to function without stopping.
  • The keyword self is used to refer to the worker. worker.postMessage('Hello'); self.postMessage('Finished'); worker.addEventListener('message', (event) => { console.log(event.data); }, false);
  • Service Workers API: This allows you to take alternative action if the network is offline, and effectively create app-like offline experiences.
