  • In a browser the global object is the window object.
  • BOM = Browser object model.
  • In BOM we find the browser being used to view a page, the dimensions of the screen it is viewed on, and which pages have been visited before the current page.
  • Every browser window, tab, popup, frame, and iframe has a window object.
  • Node.js probably won’t have a window object, but will still have a global object.
  • Global variables are variables created without using the const, let, or var keywords.
  • Method of the Number object: Number.parseInt(4.2); << 4 Number.isNaN(4.2); << false
  • Find out browser and os: window.navigator.userAgent << "Mozilla/5.0 (Macintosh;
  • Location: window.location.href << "https://www.sitepoint.com/premium/books/javascript-novice-to-ninja"

  • The search property returns a string that starts with a “?” followed by the query string parameters.
  • window.history.go(1); // goes forward 1 page window.history.go(0); // reloads the current page window.history.go(-1); // goes back 1 page
  • window.screen.height << 1024 window.screen.width << 1280
  • Cookies take the form of a text file that contain a list of name/value pairs separated by semicolons, and can be used for personalizing a user’s browsing experience, storing user preferences, keeping track of user choices (such as a shopping cart), authentication and tracking users.
  • To read the cookie (see the current contents): document.cookie: << "name=Batman; hero=true; city=Gotham"
  • Cookies expire by default when the page is closed but you can set an expiration date if you want.
  • For security reasons access to cookies is limited.
  • Cookies can be a bother. Some choose to use a library like cookies.js or jsCookies.
  • Times up!: window.setTimeout( () => alert("Time's Up!"), 3000);
  • You can use the alerts ID to stop the alert before it's finished.
  • setInterval() makes the command repeat again and again after the time you set.
  • Avoid using the "this" keyword for timing methods.
